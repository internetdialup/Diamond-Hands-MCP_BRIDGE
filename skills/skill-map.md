# Skill Map

This is the quick map for Skills in this repo. Not a giant catalog. Just enough for an agent to know what to load and when.

## repo-cognition

Use when working with the repo as operational memory: context rules, handoffs, retrieval quality, long development cycles, and AI-assisted repository governance.

This is the broad Skill. If the agent needs the general frame for how this repo thinks, start here.

## memory-context

Use when reconstructing project state, deciding what memory to load, preparing a handoff, or working with ADM/RAG context.

This is the memory retrieval Skill. It keeps the active Knob hot, old history cold, and the right slice findable.

## memory-watchdog

Use when auditing memory health: stale maps, broken references, missing Knob entries, old paths, duplicate docs, or drift.

This is the ruthless hygiene Skill. It exists so the repo does not become the thing the agent fights.
