# Project Requirement Documentation

Read through the users PRD files and create a basic PRD folder in documentation that describes the requirements of the project, the scope, and any finite details that will be pertinent to both the user and agent.

The PRD acts as the North Star that guides the project development cycle, milestones to hit, and other related information pertinent to the project for future reference.  

Cross reference the PRD with the TDD and Success Metrics to establish a consistent project development orchestration and authoring.
